📡 Wireless Protocol Comparison TEST

▼
ProtocolStatusKey Feature
WEP❌ DeprecatedWeak encryption — cracked quickly. Do NOT use.
WPA⚠ InsecureImproved over WEP but still vulnerable. Avoid.
WPA2⚠ Widely usedAES encryption, still acceptable but not ideal.
WPA3✅ Most SecureSAE replaces PSK handshake — prevents offline dictionary attacks.
WPA3 = most secure standard on the exam. WEP = deprecated due to weak encryption.

🔑 WPA3 Features TEST

▼
FeatureWhat it does
SAE (Simultaneous Authentication of Equals)Replaces WPA2 PSK handshake. Prevents offline dictionary attacks — attacker must interact with device directly to guess passwords.
128-bit encryptionPersonal networks (home use)
192-bit encryptionHigh-security / enterprise / government environments
PFS (Perfect Forward Secrecy)Compromised password cannot decrypt past traffic. Uses ephemeral (short-lived) keys.
Diffie-Hellman (DH)The only algorithm that produces ephemeral keys — what makes PFS work.
PMF (Protected Management Frames)Protects against de-authentication and dis-association attacks.
DPP / Easy ConnectDev Provisioning Protocol — securely connects IoT devices without screens using QR codes.

🔐 Port Security: Switchport, 802.1X, EAPoL TEST

▼
Remember these 3 port security types: Switchport Security → 802.1X → EAPoL
TypeScopeWhat it does
Switchport SecurityPhysical LAN (switch ports)Locks a switch port to specific MAC addresses
802.1XWired AND wirelessNetwork Access Control — authenticates users/devices via RADIUS before granting network access
EAPoL (EAP over LAN)Strictly wirelessCarries EAP authentication messages between client (supplicant) and access point (authenticator) at the data link layer

802.1X — Three Components

ComponentRole
SupplicantThe client device trying to connect
AuthenticatorThe network device controlling access (access point or switch)
Authentication ServerRADIUS server — the AAA server that verifies credentials
802.1X common in WPA2 Enterprise and WPA3 Enterprise — requires a RADIUS server.

🗺️ Site Surveys & Heat Maps

▼
  • A site survey physically maps wireless coverage — walk the space to check signal strength at different points
  • A heat map uses color to visualize signal strength — RED = hotspot (strongest), blue/green = cold (weakest)
  • Heat map color scales vary — always read the legend for that specific tool
  • Wi-Fi scans can be run without physically moving if needed, but walking the area gives better accuracy

🛡️ Wireless Hardening Best Practices

▼
  • Use WPA3 if available; WPA2 is acceptable
  • Use a complex password / passphrase
  • Disable WPS (Wi-Fi Protected Setup) — it is insecure
  • Change default router credentials
  • Keep firmware updated
  • Consider hiding or changing your SSID
  • Enable a firewall on the router
  • Use a guest network for visitors (network segmentation)
WPS = insecure, disable it. Default credentials = risk. Guest network = segmentation for security.

🔍 What Is Vulnerability Management? TEST

▼

Vulnerability management is the continuous process of identifying, evaluating, prioritizing, and remediating security weaknesses in systems, networks, and applications to reduce the risk of exploitation.

Continuous — not a one-time activity. New vulnerabilities are discovered constantly.
StepAction
1. IdentifyRun vulnerability scanners to find weaknesses
2. EvaluateCheck CVE list and CVSS score
3. PrioritizeHighest CVSS score = fix first
4. RemediatePatches, config changes, or compensating controls

🖥️ Vulnerability Scanners TEST

▼

A vulnerability scanner automatically scans systems, networks, or applications for known security weaknesses by comparing the environment against databases of known vulnerabilities (CVE list).

ScannerNotes
NmapThe "most famous" — maps your entire network, finds active devices, identifies unknown/unauthorized devices (unknown device = vulnerability)
NessusCommercial vulnerability scanner — widely used in enterprise

Vulnerability scan output includes vulnerability severity ratings — helps you prioritize what to fix first.

📊 CVSS — Common Vulnerability Scoring System TEST

▼

CVSS assigns a numeric score (0–10) to a vulnerability reflecting its severity. Use it to prioritize remediation — highest score first.

ScoreSeverityAction
0.0NONEInformational only
0.1 – 3.9LOWPatch in next scheduled cycle
4.0 – 6.9MEDIUMPrioritize and schedule remediation
7.0 – 8.9HIGHRemediate quickly
9.0 – 10.0CRITICALRemediate IMMEDIATELY
High CVSS score = fix first. Example: CVSS 8.7 = HIGH severity for SSH vulnerability (TP-Link, CVE-2026 example from notes).
How to check: Go to nvd.nist.gov → search the vulnerability → look at the CVSS score → prioritize by score.

⚖️ Vulnerability Scan vs Penetration Test

▼
ToolPurposeResult
Vulnerability ScannerFinds theoretical weaknesses in a networkList of possible vulnerabilities with severity — does NOT confirm exploitability
Penetration TestActively tries to exploit vulnerabilitiesConfirms whether a vulnerability is actually exploitable in your environment
Scanner = "you might have a problem." Pen test = "you definitely have a problem — here's the proof."

🏭 STIX & TAXII — Threat Intelligence Sharing ON TEST

How organizations describe, share, and act on cyber threat intelligence (CTI).

🌐 Dark Web & Threat Intelligence Monitoring NEW FROM NOTES

▼

The dark web exists in a semi-trusted zone (similar to a DMZ concept). Security teams monitor it for threat intelligence — not as part of normal IT work, but to stay ahead of attackers.

What orgs monitor the dark web forWhy it matters
Stolen usernames & passwords from your orgEarly warning of credential compromise before attackers use them
Leaked company dataDetect data breaches that haven't been discovered internally yet
Malware exploit kits targeting your systemsProactive defense — know what's being built against you
Individual users do NOT browse the dark web. This is handled by specialized tools and security teams only. You understand it exists and how attackers use it — you don't participate in it.

📄 STIX — Structured Threat Information eXpression TEST

▼

STIX is a standardized language / format for describing cyber threat intelligence (CTI). Think of it as the vocabulary — a common way for organizations to write down and share what they know about threats.

STIX = the FORMAT / LANGUAGE. It defines WHAT you're saying about a threat.
STIX DescribesExamples
Threat ActorsWho is attacking (nation-state, criminal group, insider)
Attack PatternsTTPs — Tactics, Techniques, and Procedures (linked to MITRE ATT&CK)
MalwareMalware families, behavior, capabilities
Indicators (IOCs)IP addresses, file hashes, domain names that indicate compromise
VulnerabilitiesCVEs and weaknesses being exploited
Courses of ActionRecommended mitigations or responses
CampaignsA group of related attacker activity over time

STIX Technical Facts

  • Format: JSON-based
  • Maintained by: OASIS (open standards organization)
  • Current version: STIX 2.x
  • Two object types: SDOs (STIX Domain Objects — the "things") and SROs (STIX Relationship Objects — how things connect)

🕒 TAXII — Trusted Automated eXchange of Intelligence Information TEST

▼

TAXII is the transport protocol / delivery mechanism for sharing STIX data. Think of it as the postal system — STIX is the letter, TAXII is how it gets delivered.

TAXII = the TRANSPORT / HOW you share it. STIX data travels over TAXII.
FeatureDetail
TransportUses HTTPS — secure, encrypted delivery
Collection modelRequest/response — clients query a TAXII server for available threat data
Channel modelPublish/subscribe — producers push updates to subscribers automatically
What it carriesPrimarily STIX objects (but technically transport-agnostic for other CTI formats)
Who uses itISACs (Information Sharing and Analysis Centers), government agencies, security vendors, enterprises

⚖ STIX vs TAXII — Side-by-Side Exam Comparison TEST

▼
Most tested concept: STIX = language/format. TAXII = transport/mechanism. They work TOGETHER.
STIXTAXII
Stands forStructured Threat Information eXpressionTrusted Automated eXchange of Intelligence Information
RoleThe FORMAT — defines what threat data looks likeThe TRANSPORT — defines how threat data is shared
AnalogyThe letter / the languageThe postal service / the delivery truck

Question cue"describes," "expresses," "format for," "language for""shares," "transports," "exchanges," "delivers," "mechanism for"
Memory trick: STIX = the STICK that writes (describes). TAXII = the TAXI that drives (transports).
