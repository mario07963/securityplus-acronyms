Asymmetric Rules — Memorize These!

  • Only the private key can decrypt data encrypted with the public key
  • Only the public key can decrypt data encrypted with the private key
  • The public key verifies but does NOT decrypt a private key's signature

📨 Attack Vectors

▼

Message-Based Attacks

  • Email (Phishing) — most common; fake emails luring victims into providing info or clicking links
  • SMS / Smishing — phishing via text message. If you see SMS on the test = Smishing!
  • Vishing — voice phishing; talking someone into doing something over a phone call

File-Based Attacks

  • Embedded macros, malicious PDFs, Office macros, Trojan installers
  • Fileless malware — no file stays on the system; a downloader pulls malware, runs it, then removes itself
  • Images can be malicious via steganography — hidden malware or exfiltrated data inside image files

Removable Media

  • USB drives, SD cards, external HDDs can exfiltrate data or introduce malware
  • Even air-gapped systems can be compromised via removable media (e.g., Stuxnet on Iranian nuclear facility)
For exam: Disable AutoRun, restrict USB devices, user training

🚨 Indicators of Compromise (IoC)

▼
  • Account Lockout — user account got locked out
  • Concurrent Sessions — same account logged in from multiple places
  • Impossible Travel — login from New York then Paris 10 minutes later
  • Resource Consumption — associated with DDoS
  • Inaccessibility — service disruption (DDoS)
  • Out-of-Cycle Logging — logs generated at unexpected times; verify timestamps match
  • Missing Logs — attacker disabled logging to cover tracks
  • Published/Documented — data exposure or leakage
  • Blocked Content — content being blocked unexpectedly

🛡️ Hardening, Patching & Endpoint Protection

▼

Hardening

  • Securing a system by reducing its attack surface — removing unnecessary things from default state
  • Disable unnecessary ports and protocols
  • Change default passwords
  • Remove unnecessary software

Patching

  • Systematic notification, identification, deployment, installation, and verification of updates
  • Updates = patches, hotfixes, service packs
  • Must be kept current to address newly discovered vulnerabilities

🔌 Port Security & Sensors

⚠ ON TEST ▼

Port Security

  • A switch feature that controls which devices are allowed to connect to specific switch ports based on MAC address
  • Administrators can allow only specific MAC addresses or limit the number of devices per port
  • If an unauthorized device is detected, the switch can: shut down the port, restrict traffic, or send an alert
  • Protects against MAC flooding and unauthorized device connections
  • Used in environments where physical access to ports cannot be fully controlled (offices, classrooms)

Sensors

  • Components that collect and monitor data from network traffic, systems, or environments to detect potential threats
  • Used in IDS, IPS, and SIEM solutions — think of them as the "eyes and ears" of your security system
  • Passive sensor — monitors traffic without interfering (like NIDS; no impact on performance)
  • Inline sensor — sits directly in the traffic path; can support prevention actions (like NIPS)

Session Affinity (Sticky Sessions)

  • Also called affinity or stickiness
  • A user's requests are consistently sent to the same server for the duration of their session
  • Achieved using cookies, IP hash, or session IDs
  • Required for apps that store user-specific data locally (login state, shopping cart)
  • Risk: If that server fails, the user's session data may be lost

IPSec — Internet Protocol Security

  • Suite of protocols that secures network communications — operates at Layer 3 (Network Layer)
  • Used for VPN tunneling — provides confidentiality, integrity, and authentication

🔀 Software-Defined Networking (SDN)

⚠ ON TEST ▼
Exam one-liner: SDN = centralized network control. Controls how data moves across networks. Think of it as network-level virtualization.

The 3 Planes of SDN — Must Know!

PlaneRoleKey Point
Control PlaneMakes decisions about how traffic should flow — handled by a central controller that determines routing paths and network policiesDecides WHERE data goes — does NOT move actual data
Data PlaneAlso called the Forwarding Plane — actually moves packets from one place to another. Exists on switches/routers that carry out control plane instructionsDoes the actual packet forwarding
Management PlaneUsed for configuring, monitoring, and managing network devicesAdmin/monitoring interface for the network
  • SDN separates the control logic from the physical hardware — traditional hardware (firewalls, switches) is virtualized
  • Provides better visibility and faster security enforcement
  • Containers control how applications run; SDN controls how data moves across networks

SD-WAN (Software-Defined Wide Area Network)

  • Creates a software-defined network over long-range links (WAN), typically between data centers
  • Each link is authenticated and secure; governed by an SD-WAN controller
  • Uses centralized controller to dynamically route traffic based on performance, latency, and priority
  • On the exam, SD-WAN is often associated with SASE

SASE & SSE — ON TEST

TermWhat It IsIncludesExam Tip
SASE (Secure Access Service Edge)Cloud-based architecture combining networking + security into one integrated solution delivered over the internetSD-WAN, Secure Web Gateway (SWG), CASB, Firewall-as-a-Service, Zero Trust Network Access (ZTNA)SASE = Networking + Security. Aligns with Zero Trust.

⚙️ Embedded Systems, ICS & SCADA

⚠ ON TEST ▼

Embedded Systems

  • An OS with a dedicated, single function — e.g., a smart thermostat, traffic light, washer/dryer
  • Traditionally built with low levels of security — users often unaware of need to change defaults or update
  • Examples of embedded platforms: Raspberry Pi, Arduino, ESP32
  • Security challenges: patch availability, inability to patch, limited compute/power, high availability requirements

ICS / SCADA / RTOS

SystemDescriptionExam Tip
ICS (Industrial Control System)Broad term for systems that manage and automate industrial processes. Includes PLCs (Programmable Logic Controllers) and DCS (Distributed Control Systems)Prioritizes uptime over security — creates risk
SCADA (Supervisory Control and Data Acquisition)A type of ICS used for remote monitoring and control across large geographic areas. Collects sensor data, sends to centralized controller, operators issue commands. Examples: power grids, water systemsSCADA = remote + large geographic area + critical infrastructure
RTOS (Real-Time Operating System)OS that processes data and responds within a strict time constraint. Designed for predictable, immediate responses. Used in aircraft autopilot, medical equipment, automotive systemsPriority = availability + timing. Security controls must NOT interfere with responsiveness
Why ICS/SCADA are vulnerable: Many were designed without security in mind. Often run on legacy systems. Common risks: weak authentication, outdated software, lack of encryption, internet exposure.
Exam focus for ICS/SCADA: Network segmentation · Strict access control · Air gap critical systems from external networks · Monitoring

Special Purpose Systems

  • Medical devices, vehicles, smart meters, UAVs (drones) — built for one job
  • Like any device, their firmware/OS can be compromised — and the data sensitivity involved can be disastrous
  • Exam emphasis: hardening, segmentation, and strict access control to protect them

Embedded Architecture Considerations (exam list)

  • Patch availability vs inability to patch
  • Power and compute constraints
  • Availability and resilience requirements
  • Cost limitations
  • Responsiveness (real-time requirements)
  • Scalability and ease of deployment/recovery

🚪 Fail Open vs Fail Closed

⚠ ON TEST ▼
Fail ClosedFail Open
What happens on failureDenies all access — defaults to a secure, blocked stateAllows all access — system continues to pass traffic
SecurityMore secure — prevents unauthorized access during failureLess secure — unauthorized access may be allowed during failure
AvailabilityLower — legitimate users may also be locked outHigher — traffic keeps flowing
Why orgs prefer itHigh-security environments (military, healthcare)Business prefers it — money stops flowing when traffic stops; compensating controls should cover the gap
Exam tip: Fail Closed = more secure, blocks everything. Fail Open = more available, allows everything. If the question says the priority is traffic must always flow — choose Fail Open, even if it feels wrong. Organizations justify it by having other compensating controls in place.

💾 Backups, Replication & Snapshots

⚠ ON TEST ▼

Backup Types

TypeWhat Is Backed UpBackup TimeRestore Time
Full BackupALL data every time — starting point for other backupsHigh (slowest)Low (fastest) — all data in one set
Incremental BackupOnly changes since the last backup of any kindLow (fastest)High (slowest) — need full + every incremental in order
Differential BackupOnly changes since the last full backupMediumMedium — need full + most recent differential only

Off-Site Backups

  • Store backups in an entirely separate geographic location — critical if a disaster (fire, flood) affects the main site
  • Backup systems must have the same level of security as primary systems — they contain a complete copy of sensitive data

Snapshots

  • A saved state of operation — not exactly a backup
  • Does not matter what state the system was in before being shut down
  • Used for quick rollbacks; not a substitute for a full backup strategy

Replication

TypeHow It WorksUse Case
SynchronousCopies data simultaneously across two peered systems — changes in one are immediately reflected in the otherHigh-availability environments; zero data loss
AsynchronousCopies data from primary to a secondary location — changes are reflected for safekeeping and failoverDR sites; some data loss possible

Asset management is the lifecycle process of tracking, controlling, and eventually retiring organizational assets.
ConceptDefinition

Asset AssignmentFormally assigning an asset to a specific user, team, or department — establishes who is responsible for the asset

📋 Asset Tracking & Monitoring ▼
TermWhat It Does
EnumerationIdentifies and lists networked assets that are actively communicating on the network — automated scanning process
InventoryOrganizes and documents all physical assets owned by the organization — typically a manually maintained or CMDB-backed list
TrackingProvides ongoing visibility into where assets are, who has them, and their current status throughout the lifecycle

Inventory vs. Enumeration

  • Inventory = what the org thinks it owns (physical record)
  • Enumeration = what is actually on the network (discovered dynamically)
  • If enumeration finds a device not in the inventory → investigate immediately — could be unauthorized/rogue device
Exam Tip: Enumeration ≠ Inventory. If something shows up in enumeration but NOT in inventory, treat it as a potential security incident.
🗑️ Decommissioning & Data Disposal ▼

Decommissioning by Device Type

Device TypeRecommended Decommissioning Method
Spinning Disk (HDD)Degauss (destroy magnetic field) then shred (physical destruction)
SSD / Flash StorageSecure Erase (built-in firmware command) then shred (physical destruction) — degaussing does NOT work on SSDs
Network AppliancesReset to factory defaults and purge configuration data before disposal
Printers / ScannersFollow manufacturer guidelines — many store data internally; memory must be wiped per vendor instructions

Data Sanitization Methods

MethodDescription
Overwriting / ClearingOverwrites data with 1s, 0s, or random patterns — data is logically removed but media is reusable
PurgingMore thorough than clearing — uses techniques like Block Erase or Cryptographic Erase; media may be reusable
Crypto-EraseDestroys the encryption key for encrypted data — data is unrecoverable without the key; fast and effective
DegaussingExposes magnetic media to a strong magnetic field — destroys data and typically destroys the media too (not usable after)

Destruction Methods

MethodDescription
ShreddingPhysically shreds the media into small pieces — cross-cut or micro-cut preferred
PulverizingCrushes media into fine particles — higher assurance than shredding
IncinerationBurns media to ash — highest level of physical destruction; requires chain-of-custody documentation
DrillingPhysically drilling through platters/chips — renders media unreadable

Certification & Data Retention

  • Certification — Formal documentation that proves destruction occurred; required for compliance and audit purposes. Issued by the party performing the destruction.
  • Data Retention Policy — Before disposing of any asset, verify regulatory requirements for how long data must be kept. Some regulations (HIPAA, PCI-DSS, GDPR) mandate minimum retention periods before destruction is allowed.
Exam Tip: Degaussing works on HDDs but NOT SSDs. Crypto-erase is fastest for encrypted drives. Always get a certificate of destruction for compliance. Check retention policy BEFORE disposal.
